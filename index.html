HTML

  • HTLM is used to create the stucture of a webpage.
  • The head element containts the metadata for our web application. Metadata is data about our webpage that the browser uses to effectively serve our webpage to users. Metadata is typically not seen by users, but it contains important information about accessibility, search engines, and performance.
  • The body element represents the visible content shown to the user.
  • The title element (in the head element) defines the title for a webpage. This element is required and is extremely important for Search Engine Optimization (SEO).
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • The script element embeds client-side scripts in the HTML document.
  • Description of index.html:
  • DOCTYPE html (We see the following declaration, which tells the browser that this document is written in HTML).
  • html lang="en" (which lets the browser know that this page will be in English:).
  • The charset element sets our character encoding.
  • The http-equiv element sets an HTTP header for content.
  • The viewport element sets the default view for our webpage.

GIT

  • git status: checks what branch we are currently on.
  • git checkout main: To switch to the main branch.
  • git checkout -b branch-name: create a new branch and switches to it.
  • git add -A: Add file(new modified, deleted) to the stage.
  • git add .: Add file(new and modified) to the stage.
  • git add -u: Add file(modified and deleted) to the stage.
  • git commit -m "comment": To commit the change.
  • git pull origin main: To pull the last code from the githup to insure that we have a latest commit and changes.
  • git push origin branch-name: To publish the change to githup.
  • Click on create pull request, on githup: To open a pull request.
  • click on confirm merge: To merge the branch to the main.
